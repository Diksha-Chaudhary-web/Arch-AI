# ArchFlow AI — Agent Instructions

## Application Building Context

Read the following files in order before implementing
or making any architectural decision:

1. `context/project-overview.md`
   — product definition, goals, features, workflows, and scope

2. `context/architecture-context.md`
   — system architecture, stack, storage model, boundaries, and invariants

3. `context/ui-context.md`
   — theme, colors, typography, workspace layout, and component conventions

4. `context/code-standards.md`
   — implementation rules, TypeScript standards, API conventions, and file organization

5. `context/ai-workflow-rules.md`
   — development workflow, scoping rules, implementation boundaries, and delivery process

6. `context/progress-tracker.md`
   — current implementation phase, completed work, open questions, and next steps

---

Update `context/progress-tracker.md`
after each meaningful implementation change.

If implementation changes the architecture, scope,
workflow, or standards documented in the context files,
update the relevant file before continuing.
